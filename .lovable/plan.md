## What We Should Build (Prioritized)

### Priority 1 — Buildable Now (data exists in our tables)

These reports can be created using existing `phorest_transaction_items`, `phorest_appointments`, and `phorest_clients` data:

| Report | What It Shows | Data Source |
|---|---|---|
| **Discounts & Promotions** | Discount amount by type, staff who discount most, promo usage | `phorest_transaction_items.discount`, `promotion_id` |
| **Value of Future Appointments** | Revenue pipeline from booked-but-not-completed appointments | `phorest_appointments` where `appointment_date > now()` |
| **Top Clients** (enhanced) | Ranked by spend with visit frequency, avg ticket, services breakdown | `phorest_transaction_items` grouped by client |
| **Client Birthdays** | Upcoming birthdays for marketing outreach | `phorest_clients.date_of_birth` |
| **New Client Source** | Where new clients came from (referral, online, walk-in) | `phorest_clients.source` field |
| **Duplicate Clients** | Potential duplicates by name/email/phone match | `phorest_clients` fuzzy matching |
| **No Shows & Cancellations** (enhanced) | Combined report with revenue impact, repeat offenders | `phorest_appointments.status` |
| **Deleted Appointments** | Audit trail of removed appointments | `phorest_appointments` with deleted status if tracked |
| **Staff Transaction Detail** | Line-item detail per stylist per day | `phorest_transaction_items` by staff + date |

### Priority 2 — Buildable With Minor Schema Additions

| Report | What's Needed |
|---|---|
| **Gift Card Sold / Redeemed / Outstanding** | Need gift card tracking table (or pull from transaction items with `item_type = 'voucher'`) |
| **Online Bookings** | Need booking source field on appointments |
| **Client Courses/Packages** | Need packages/courses table |
| **Accounts Outstanding** | Need client accounts receivable tracking |

### Priority 3 — Requires New Data Collection Infrastructure

| Report | What's Needed |
|---|---|
| **Staff Clock In-Out / Hours Worked** | Time-clock integration or manual entry system |
| **Inventory List / Valuation / Reconciliation / Low Stock / Orders** | Full inventory management module (Color Bar has partial) |
| **Room/Machine Utilization** | Resource booking system |
| **Petty Cash / Cash Drawer Audit** | Cash management module |

### Consolidation Opportunities

Rather than replicating Phorest's fragmented approach (23 staff reports!), Zura should consolidate intelligently:

1. **Staff Performance Suite** — Merge Phorest's 6 "Staff Performance" variants into our existing Individual Staff Report (already comprehensive)
2. **Commission Suite** — Our Commission Report + Payroll Summary already covers Phorest's Commission Detail + Overview + Basic Wages
3. **Sales Analysis** — Our Service Category Mix consolidates Phorest's Sales by Category + Sales by Department + Top Services
4. **Client Intelligence** — Create one "Client Intelligence Report" that covers Top Clients + New Client Source + Super Clients (Referrals) + Duplicate detection in a single exportable view

## Implementation Plan

### Phase 1: Quick Wins (9 reports, data ready)
Build the Priority 1 reports listed above. Estimated: 9 new report components, 5 new hooks. All use existing transaction/appointment/client data.

### Phase 2: Gift Cards + Packages (4 reports)
Requires either mining existing transaction data for voucher-type items or adding a lightweight gift card tracking table.

### Phase 3: Infrastructure Reports (deferred)
Inventory, time-clock, and cash management reports depend on those operational modules being built first. These are roadmap items, not immediate gaps.
